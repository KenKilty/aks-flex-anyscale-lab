This lab starts with one AKS cluster and one practical capacity problem: the GPU
you need may live in another Azure region, in your datacenter, or in another
cloud environment. **AKS Flex Node** lets the cluster use that Linux compute as
worker capacity. **Anyscale on Azure** adds the managed Ray layer for Jobs,
compute profiles, and workload observability. Together, they let you run Ray
AI/ML workloads where your compute and GPUs already are.

The Azure-region path in the modules is the reference run. The same join model can
apply to reachable on-premises machines or hosts in another environment. The lab
keeps the student path narrow enough to prove placement, understand the moving
parts, and tear everything down cleanly.

## What You Will Learn

You will learn why Flex Node matters when useful compute sits outside the home
AKS region, then attach CPU or GPU hosts from a second Azure region to a single
AKS cluster. After Anyscale on Azure submits Ray Jobs onto that combined capacity,
you will prove placement with job output, Kubernetes pod data, and teardown checks.
