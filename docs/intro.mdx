This lab teaches you how to extend a single AKS cluster across Azure regions,
on-premises machines, or other clouds using **AKS Flex Node**, then run
distributed GPU workloads through **Anyscale** — without modifying your workload
code.

AKS Flex Node lets you join any Linux host to an existing AKS cluster as a
worker node. Combined with Anyscale's ability to schedule Ray Train and DeepSpeed
jobs across all available GPU capacity, this creates a unified distributed AI
platform that can span datacenter boundaries.

## What you will learn

- How AKS Flex Node works and when to use it instead of standard node pools
- How to attach GPU hosts from a second Azure region, on-premises, or another
  cloud to a single AKS cluster
- How Anyscale dispatches distributed training jobs across heterogeneous node
  locations without code changes
- How to produce machine-readable proof that your workload actually ran on nodes
  in both locations
